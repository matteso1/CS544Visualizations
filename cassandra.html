Figure I · Replication and the ring walk

The Cassandra ring.

Every key in a Cassandra cluster lives in exactly one place — at first. The ring decides where, and the replication factor decides how many more places it should also live.

A Cassandra cluster is laid out as a token ring. Each physical node owns a number of vnodes (virtual nodes), and each vnode is assigned a token — a point on the ring. When a row is written, its partition key is hashed to produce a token, and the row is sent to the first vnode whose token is greater than or equal to the key’s token, walking clockwise. That vnode’s node is the primary replica. The next RF − 1 distinct nodes clockwise hold additional replicas.

§ I.1What just happened

The cluster has four physical nodes (A, B, C, D) and eight vnodes arranged around the ring. When you drag the key, its hash-token changes; the figure walks clockwise from that token until it meets a vnode. The first RF distinct nodes it touches — skipping additional vnodes that belong to a node it has already counted — become the replica set.

The walk is always clockwise

The single most common exam mistake is walking the wrong way. The ring is ordered by increasing token, so the next vnode after a key is always the one with the smallest token that is still ≥ the key’s token. Walk counter-clockwise and you will get a different owner every time.

Replicas count distinct nodes, not vnodes

If two adjacent vnodes belong to the same node, they count as one replica. The walk simply continues past the duplicate. This is why a cluster needs RF distinct physical nodes to satisfy any RF; a 3-node cluster cannot provide RF = 5.

Quorum is a function of RF only

QUORUM = ⌊RF ÷ 2⌋ + 1. So for RF = 3, quorum is 2; for RF = 5, quorum is 3; for RF = 6, quorum is 4 (a frequent trap — half of 6 is 3, but the quorum is one more than half).

R + W > RF means strong consistency

If the write touches W replicas and the read touches R replicas and R + W > RF, then any read’s replica set must overlap with the write’s replica set in at least one node — so the read is guaranteed to see the write. That overlap is what “strong consistency” means in Cassandra.

§ I.2The rules to remember

walk            : clockwise, increasing token
owner           : first vnode with token ≥ key's token
replicas        : next RF distinct nodes, clockwise
QUORUM(RF)      : floor(RF / 2) + 1
strong consist. : R + W > RF
